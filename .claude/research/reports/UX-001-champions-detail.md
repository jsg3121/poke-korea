# UX-001: 챔피언스 포켓몬 상세 페이지 UI/UX 개선 분석

- 조사일: 2026-04-19
- 에이전트: ux-designer
- 분석 대상: `/champions/list/[pokemonId]`
- 분석 방법: 코드 분석 + 경쟁 서비스 벤치마킹 (Pikalytics, Smogon, OP.GG)

---

## 1. 현재 UI/UX 문제점 분석

### 1-1. 정보 계층 구조 문제

**문제 A: 스탯이 숫자 나열에 그침 (모바일)**

모바일 컨테이너(`ChampionsDetail.container.tsx`)의 스탯 섹션은 `dl > dt/dd` 4열 그리드를 사용한다. 체력/공격/방어/스피드를 1행에, 특공/특방/합계를 2행에 배치하는 구조다. 사용자는 각 수치가 높은지 낮은지 맥락 없이 숫자만 보게 되므로, 이 포켓몬의 스탯 강점을 직관적으로 파악하기 어렵다.

- 체력 체계: 포켓몬 스탯은 최솟값(~1)부터 최댓값(~255) 범위를 가진다. 숫자만 제시하면 사용자가 상대적 강약을 즉시 판단하기 어렵다.
- 합계 수치 역시 강조 처리(`font-bold`)는 되어 있으나, 다른 포켓몬 대비 높은지 낮은지 알 수 없다.

**문제 B: 메타 핵심 지표가 시각적으로 묻힘**

`ChampionsMetaSection`에서 사용률과 승률은 `text-2xl font-bold`로 표시되지만, 티어 배지(`w-16 h-16`)와 나란히 배치되어 시각적 무게가 분산된다. 사용자가 페이지에 진입했을 때 "이 포켓몬이 얼마나 강한가"를 즉각 인지할 수 있는 핵심 지표 영역이 없다.

**문제 C: 기술/아이템/특성 섹션의 정보 밀도가 낮음**

`ChampionsMetaList` 컴포넌트는 이름과 사용률(%)만 텍스트 나열 형태로 보여준다. 상위 항목과 하위 항목의 차이가 숫자로만 구별되기 때문에, 어느 항목이 압도적으로 주류인지 시각적으로 파악하는 데 시간이 걸린다. 예를 들어 1위 기술이 90%이고 2위가 30%라면 이 차이가 얼마나 큰지 텍스트로는 전달되지 않는다.

**문제 D: 파트너 섹션과 다른 메타 섹션이 시각적으로 동일 취급됨**

`ChampionsPartnerList`는 이미지(24x24)를 포함하지만, `ChampionsMetaList`와 동일한 카드 크기와 배경색(`bg-gray-50`)을 사용하므로 두 섹션의 정보 성격 차이(아이템/기술 vs 포켓몬 파트너)가 시각적으로 구분되지 않는다.

### 1-2. 시각적 표현의 한계

**문제 E: 포켓몬 이미지 크기가 작음 (데스크톱)**

데스크톱에서 포켓몬 이미지는 `192x192`(w-48 h-48)이고, 좌측 사이드 카드(`w-72`)에 배치된다. 데스크톱 해상도에서 상세 페이지임에도 목록 카드와 거의 동일한 이미지 크기를 사용하므로, 상세 페이지로서의 볼륨감이 부족하다.

**문제 F: 타입별 배경 그라디언트가 카드 영역에만 적용됨**

포켓몬 타입 색상 그라디언트가 기본 정보 카드(`article`)에만 적용되고, 메타 섹션은 흰색/회색 배경이다. 타입 컬러 아이덴티티가 페이지 전체에 일관성 있게 유지되지 않아 디자인적 통일감이 떨어진다.

**문제 G: 빈 상태(meta null) 처리가 단조로움**

`ChampionsMetaSection`에서 meta가 null일 때 단순 텍스트(`메타 데이터를 불러올 수 없습니다.`)만 표시된다. 사용자가 다음에 무엇을 해야 할지 안내가 없다.

**문제 H: isStale 경고 배너가 액션 없이 경고만 존재**

`meta.isStale` 상태일 때 노란색 배너를 노출하지만, "언제 업데이트될 예정인지" 혹은 "다른 시즌 데이터 보기" 같은 사용자 행동 유도가 없다.

### 1-3. 사용자 흐름 개선점

**문제 I: 상세 페이지에서 목록으로 돌아가는 내비게이션이 없음**

현재 뷰 코드(`ChampionsDetail.mobile.tsx`, `ChampionsDetail.desktop.tsx`)에는 뒤로가기 링크나 브레드크럼이 없다. 사용자가 목록으로 돌아가려면 브라우저 뒤로가기에 의존해야 한다.

**문제 J: 파트너 포켓몬에 클릭 인터랙션이 없음**

`ChampionsPartnerList`의 파트너 포켓몬은 이미지와 이름이 표시되지만 해당 포켓몬의 상세 페이지로 이동하는 링크가 없다. 파트너 포켓몬에 관심을 가진 사용자가 추가 탐색을 할 수 없다.

**문제 K: 모바일에서 스탯 영역이 불필요하게 4열 구성**

모바일에서 4열 그리드(`grid-cols-4`)를 사용하면 개별 항목 너비가 매우 좁아진다. 375px 기준으로 각 컬럼은 약 70px에 불과하여 가독성이 떨어질 수 있다.

## 2. 경쟁 서비스 벤치마킹

### 2-1. Pikalytics (pikalytics.com/pokedex/championstournaments)

포켓몬 챔피언스 전용 메타 데이터를 제공하는 가장 직접적인 경쟁 서비스다.

**레이아웃 구조:**

- 피라미드 정보 계층: 포켓몬 이미지/기본 정보 → 핵심 지표(사용률, 랭킹) → 세부 메타 데이터(기술/아이템/특성) → 실전 팀 사례
- 사용률 순위(`Monthly Rank #1`)와 사용률 퍼센트(`Usage 54%`)를 헤더 영역에 큰 숫자로 강조
- 각 기술/아이템/특성 항목마다 수치를 우측 정렬로 나열 (내림차순)
- 파트너 포켓몬: 이미지 + 이름 + 타입 + 사용 비율 + 클릭 가능한 링크

**시각화 패턴:**

- 백분율 수치는 텍스트 나열 방식을 사용하되, 상위 항목이 명확히 구별되도록 폰트 가중치 차이를 적용
- 기술 섹션이 가장 방대하게 구성 (6~12개 항목)
- EV 스프레드 섹션을 별도로 제공 (포켓몬 능력치 배분 정보)
- 실전 팀 구성 사례를 2×3 그리드로 표시하며 플레이어명 + 전적 포함

**시사점:** 데이터 중심 레이아웃, 파트너 포켓몬에 대한 상호작용(링크), 실전 팀 사례 제공이 강점이다.

### 2-2. Smogon Strategy Pokedex (smogon.com/dex)

경쟁 포켓몬 분석 분야의 레퍼런스 서비스다.

**레이아웃 구조:**

- 상단에 포켓몬 이미지 + 기본 스탯 + 타입 정보를 한 영역에 집약
- 분석 텍스트(Overview) → 세트 구성(Set) → Set Comments → Other Options 순의 단계적 정보 공개
- 스탯은 바 형태로 시각화하여 상대적 강약을 직관적으로 표시

**시사점:** 스탯 바 시각화와 단계적 정보 공개 구조가 학습에 효과적이다.

### 2-3. OP.GG 챔피언 통계 (op.gg/lol/statistics/champions)

게임 메타 정보 사이트의 UI 레퍼런스다.

**레이아웃 패턴:**

- 핵심 지표(승률, 픽률, 밴률)를 카드 상단에 대형 수치로 표시
- 이전 패치 대비 변화율(+/-) 표시로 트렌드 정보 제공
- 스킬/룬 사용률을 진행 바(progress bar) 형태로 시각화
- 카운터 픽/시너지 정보를 포켓몬 이미지와 함께 그리드 표시

**시사점:** 트렌드 정보, 진행 바 시각화, 카운터/시너지 정보 구조가 메타 분석 도구로서의 유용성을 높인다.

### 2-4. 포케샘플 (pokesample.com/ko/statistics)

국내 포켓몬 메타 정보 서비스로 직접적인 경쟁 서비스다.

**시사점:** 실시간 사용률, 채용 기술, 아이템 통계를 빠르게 파악할 수 있는 UI가 핵심 가치다. 한국어 사용자를 대상으로 하는 서비스로, 포케코리아와 동일한 타깃을 공유한다.

## 3. 개선 방향 제안

### 3-1. 레이아웃 개선안

**Hero 섹션 강화 (모바일/데스크톱 공통)**

현재 기본 정보 카드와 메타 섹션이 단순 상하 배치로 분리되어 있다. 제안하는 구조는 다음과 같다.

```
[Hero 영역] — 포켓몬 이미지(대형) + 이름 + 번호 + 타입 태그 + 티어 배지
                타입 그라디언트 배경 전체 영역에 적용
[핵심 지표 바] — 사용률 / 승률 / 티어를 수평 3분할 카드로 강조
[스탯 섹션] — 바 시각화 포함
[메타 데이터 섹션] — 기술 / 아이템 / 특성 / 파트너 카드 그리드
[브레드크럼 + 뒤로가기] — 상단 고정
```

**데스크톱 2열 구조 개선**

현재 데스크톱은 좌측 `w-72` 고정 카드 + 우측 `flex-1` 메타 섹션이다. 개선안은 다음과 같다.

- 좌측 컬럼을 `w-80`으로 넓히고 포켓몬 이미지를 `256x256`으로 확대
- 스탯 바 시각화를 좌측 컬럼에 포함
- 우측 컬럼을 탭 구조로 변경: [기술] [아이템] [특성] [파트너] 탭으로 내용 밀도 높이기

**모바일 레이아웃 개선**

- 4열 스탯 그리드 → 2열 그리드 (레이블 + 바 + 수치)로 변경
- 스크롤 영역을 최소화하기 위해 기술/아이템/특성을 수평 스와이프 가능한 탭으로 구성

### 3-2. 스탯 시각화 방안

**진행 바(Progress Bar) 도입**

포켓몬 스탯 최댓값을 255로 기준 잡고, 각 스탯 수치를 비율로 환산하여 바로 표시한다.

```
체력   [████████████░░░░░░] 105 / 255
공격   [████████░░░░░░░░░░] 80 / 255
방어   [█████░░░░░░░░░░░░░] 55 / 255
...
```

- 바 색상은 수치에 따라 3단계로 구분: 낮음(gray), 보통(blue), 높음(타입 메인 색상)
- 합계 수치는 별도 강조 영역으로 분리하여 한눈에 인지할 수 있도록 표시
- 구현 시 `div`의 `width` CSS를 퍼센트로 설정하면 Chart.js 없이도 Tailwind 유틸리티만으로 구현 가능

**Why:** 스탯 바 시각화는 Nielsen Norman Group의 데이터 시각화 가이드라인에서 수치 데이터를 비교할 때 막대 차트가 숫자 나열보다 빠른 인지를 돕는다고 명시하고 있다. 포켓몬 스탯처럼 6개 항목을 비교할 때 특히 유효하다.

### 3-3. 메타 정보 표현 방식 개선

**사용률/채용률 바 도입 (ChampionsMetaList)**

현재 텍스트 + 퍼센트 나열 방식을 진행 바 + 퍼센트로 변경한다.

```
[인기 기술]
번개 펀치  [████████████████░░] 90.2%
칼춤       [████████░░░░░░░░░░] 45.1%
고속이동   [████░░░░░░░░░░░░░░] 22.8%
```

- 1위 항목이 시각적으로 압도적임을 즉시 인지 가능
- 바 너비는 가장 높은 항목을 100%로 기준 삼아 상대 비율로 표시 (절대 비율보다 섹션 내 상대 비율이 더 의미 있음)

**파트너 포켓몬 카드 링크 연결**

`ChampionsPartnerList`의 각 파트너 항목을 `/champions/list/[pokemonId]` 링크로 감싸 클릭 시 해당 포켓몬 상세 페이지로 이동하도록 한다. 파트너 포켓몬 이미지 크기를 `40x40`으로 확대하여 인지성을 높인다.

**isStale 경고 배너 개선**

단순 경고 텍스트에서 액션 포함 배너로 변경한다.

```
[!] 이 데이터는 이전 시즌 기준입니다. 최신 업데이트: 2026-03-15
```

### 3-4. 반응형 전략

| 브레이크포인트              | 레이아웃 전략                                  |
| --------------------------- | ---------------------------------------------- |
| 모바일 (max-width: 768px)   | 단일 컬럼, 풀 너비 Hero, 탭으로 메타 정보 전환 |
| 데스크톱 (min-width: 769px) | 좌측 고정 정보 카드 + 우측 메타 섹션 2열       |

**모바일 특화 개선:**

- Hero 영역 높이: 포켓몬 이미지를 `200x200`으로 확대, 타입 그라디언트 배경을 전체 너비로
- 기술/아이템/특성/파트너를 수평 탭(`overflow-x: auto`, 스냅 스크롤)으로 구성하여 스크롤 깊이 감소
- 브레드크럼을 `sticky top-0`으로 고정하여 긴 페이지에서도 위치 파악 용이

**데스크톱 특화 개선:**

- 좌측 카드: 이미지 256x256, 스탯 바 시각화 포함, 고정 너비 `w-80`
- 우측 메타 섹션: 기술/아이템을 2열로, 파트너를 별도 그리드(3열)로 배치

### 3-5. 인터랙션 패턴

**내비게이션:**

- 상단에 브레드크럼 `챔피언스 > 도감 > [포켓몬 이름]` 추가
- 데스크톱에서 좌우 인접 포켓몬으로 이동하는 화살표 버튼 추가

**로딩 상태:**

- 스탯 바는 0에서 실제 값까지 CSS transition으로 슬라이드인 애니메이션 적용
- 메타 섹션은 스켈레톤 UI로 대체

**에러/빈 상태:**

- meta null 상태: "아직 메타 데이터가 없습니다. 시즌이 시작되면 업데이트됩니다." + 목록으로 돌아가기 CTA
- 이미지 로드 실패: 타입 색상 배경 + 포켓몬 번호 텍스트 폴백

## 4. 우선순위 정리

우선순위는 구현 난이도(Low/Mid/High)와 사용자 임팩트(Low/Mid/High)를 기준으로 정리한다.

| 순위 | 개선 항목                                | 구현 난이도 | 사용자 임팩트 | 근거                                                |
| ---- | ---------------------------------------- | ----------- | ------------- | --------------------------------------------------- |
| 1    | 파트너 포켓몬 클릭 링크 연결             | Low         | High          | 추가 탐색 유도, 단순 Link 컴포넌트 추가만으로 해결  |
| 2    | 브레드크럼 / 뒤로가기 내비게이션 추가    | Low         | High          | 사용자 방향 감각 제공, 표준 내비게이션 패턴         |
| 3    | 스탯 바 시각화 도입                      | Mid         | High          | 숫자 → 시각적 비교, Tailwind로 구현 가능            |
| 4    | 기술/아이템/특성 채용률 바 시각화        | Mid         | High          | 주류 채택 항목 즉각 인지, 동일 컴포넌트 패턴 재사용 |
| 5    | 모바일 스탯 그리드 4열 → 2열 변경        | Low         | Mid           | 가독성 직접 개선, CSS 수정만으로 해결               |
| 6    | isStale 배너에 업데이트 시점 정보 추가   | Low         | Mid           | 사용자 신뢰도 향상                                  |
| 7    | 모바일 메타 섹션 탭 구조 전환            | Mid         | Mid           | 스크롤 깊이 감소, React 상태 관리 필요              |
| 8    | 데스크톱 포켓몬 이미지 256x256 확대      | Low         | Mid           | 상세 페이지 볼륨감 향상, 단순 크기 변경             |
| 9    | 타입 그라디언트 배경 Hero 전체 영역 적용 | Mid         | Mid           | 시각적 아이덴티티 강화                              |
| 10   | meta null 빈 상태 개선 (CTA 포함)        | Low         | Low           | 엣지 케이스 경험 개선                               |

## 5. 핵심 요약

### 즉시 적용 가능한 개선 (Low 난이도, High 임팩트)

1. 파트너 포켓몬에 `/champions/list/[pokemonId]` 링크 연결
2. 뷰 상단에 브레드크럼 내비게이션 추가
3. 모바일 스탯 그리드를 4열에서 2열로 변경

### 중기 개선 (스프린트 단위로 진행 권장)

1. 스탯 바 시각화 - Tailwind `width` 유틸리티로 Chart.js 없이 구현 가능
2. 기술/아이템/특성 채용률 바 시각화 - `ChampionsMetaList` 컴포넌트 개선
3. 데스크톱 포켓몬 이미지 크기 256x256으로 확대

### 가장 큰 경쟁 우위 차별점

Pikalytics는 실전 팀 사례(Top Teams)를 상세 페이지에서 연계하는 반면, 현재 포케코리아 상세 페이지는 메타 데이터 확인에서 끝난다. 파트너 포켓몬 링크 연결을 시작으로 사용자가 상세 페이지에서 팀 빌딩까지 이어지는 흐름을 설계하는 것이 중장기적으로 중요한 방향이다.
