## 1. Executive Summary

포케코리아는 한국어 전용 포켓몬 도구 사이트로서 구글 SEO 구현 수준은 양호하나, 한국 검색 시장 점유율 62.86%를 차지하는 네이버에서의 유입(36%)이 잠재 대비 크게 저조하다. 전체 MAU 23,047명 중 /type-effectiveness 단일 페이지가 26,228 UV를 차지하는 구조는 트래픽 집중 위험과 동시에 이 키워드군에서의 강점을 방증한다. **OP.GG가 포켓몬 챔피언스 전용 서비스를 한국어로 런칭함에 따라 핵심 도구 키워드 경쟁이 본격화**됐으나, OP.GG의 본편(Z-A, 스칼렛/바이올렛) 미커버 및 **챔피언스 모바일 출시(2026 여름)는 단기 내 가장 큰 트래픽 성장 기회**다. 내부 기술 역량(Next.js App Router, SSR/ISR, 구조화 데이터)은 충분히 갖춰져 있으며 **실행 속도가 경쟁 우위 결정 요소**다.

## 2. 현재 포지셔닝 평가

```text
                 도구/계산기 특화
                       ↑
  OP.GG(챔피언스) |  포케코리아(현재)  |  yakkun
                  |                  |
  정보 위키 ←─────┼──────────────────┼──→ 대전 메타
                  |                  |
  나무위키        |  포케모음         |  포케샘플
                       ↓
                 정보/데이터 특화
```

**위치 평가**: "도구 특화 + 종합 커버리지" 영역에서 한국어 시장 내 유일한 사이트. 단, OP.GG 진입으로 챔피언스 대전 도구 영역에서의 독점이 해소됐다. 나무위키가 정보 키워드를 장악하고 있어 "도구성 쿼리" 집중이 불가피하며, 이는 현재 트래픽 구조(/type-effectiveness 단일 집중)와 일치한다.

## 3. 강점 (Strengths)

### S1. 구글 SEO 구현 완성도 — 점수 4/5

**근거 (코드)**:

- 전 페이지 `generateMetadata` / `metadata` export 구현. `/type-effectiveness`, `/list`, `/moves`, `/ability`, `/quiz`, `/champions/*` 모두 페이지별 고유 title/description/canonical/OG 태그 적용 (`src/app/type-effectiveness/_metadata/typeEffectivenessMetadata.ts`, `src/app/champions/_metadata/championsMetadata.ts` 등)

## 4. 약점 (Weaknesses)

### W1. CTR 2.29% — 검색 순위 대비 심각한 이탈 — 점수 2/5

**근거 (지표)**: 평균 검색 순위 6.8임에도 CTR 2.29%는 정상 범위(순위 7위 기준 약 3~5%) 하단. 노출 99,609회 대비 클릭 2,281회. 순위 3위 달성 시 CTR 15~30% 도달 가능(MI 보고서 추정).

**근거 (코드)**: 홈 메타 description "1025마리 포켓몬 도감, 타입 상성 계산기, 800개 이상 기술 정보, 300개 이상 특성 정보, 매일 새로운 포켓몬 퀴즈! 빠르고 정확한 포켓몬 백과사전."(`src/app/_metadata/homeMetadata.ts`)는 길이가 적절하나, 네이버 기준 권장(60~80자) 검토 필요. layout.tsx 기본 description "언제, 어디서든, 포켓몬의 정보를 빠르고 편리하게 확인하실 수 있습니다."는 개별 페이지 override 없는 경우 표출되며, 경쟁력 낮음.

### W2. /type-effectiveness 단일 페이지 의존 — 점수 2/5

**근거 (지표)**: /type-effectiveness 30일 UV 26,228이 사이트 전체 MAU 23,047 초과. 다수 방문자가 이 페이지만 보고 이탈. 2위 /list(8,948), 3위 /moves(8,389)와 격차가 크다.

**근거 (코드)**: 홈(`src/app/page.tsx`)은 `force-dynamic`으로 SSR. 퀴즈, 오늘의 포켓몬 노출하나 인터링킹(타입 상성 → 포켓몬 상세 → 기술 등)을 강제하는 UI 흐름이 코드 레벨에서 미확인. ChampionsHomeContainer, HomeBannerContainer 모두 카드 나열 구조로, 다음 행동 유도 CTA 미확인.

### W3. 챔피언스 상세 페이지 메타 description 동일 패턴 — 점수 2/5

> **v3 수정 (2026-05-04)**: 초기 진단(상세 페이지 부재)은 잘못이었음. 실제 페이지는 `src/app/champions/list/[pokemonId]/page.tsx`로 완전 구현되어 있음. 진짜 문제는 메타 description의 boilerplate 패턴이며 이를 새로 진단함.

**근거 (지표)**: `/champions` 계열 30일 합계 3,494명. 페이지가 구현되어 있고 사이트맵에도 등록되었음에도 노출이 저조함은 **메타 품질 + 인덱싱 가속화 부족**으로 추정.

**근거 (코드, v3)**:

- 페이지 파일 ✅ 존재: `src/app/champions/list/[pokemonId]/page.tsx`
- 메타데이터 ✅ 존재: `src/app/champions/list/[pokemonId]/_metadata/generateChampionsDetailMetadata.ts`
- 사이트맵 동적 등록 ✅: `GetChampionsPokemonListDocument`로 전체 챔피언스 포켓몬 목록 가져와 URL 자동 생성
- BreadcrumbList JSON-LD ✅, OG/Twitter/canonical/robots ✅
- desktop/mobile 분리 ✅: `ChampionsDetail.desktop.tsx` / `ChampionsDetail.mobile.tsx`
- ISR 24시간 ✅: `revalidate = 86400`
- 본편 detail 페이지로 인터링크 ✅: `ChampionsDetailContainer`에 `/detail/{pokemonNumber}` 링크 구현

**진짜 문제 (v3 새 진단)**:

1. **메타 description 동일 패턴**: `generateChampionsDetailMetadata.ts:76` 에서 `${pokemon.name}의 포켓몬 챔피언스 사용률, 인기 기술, 아이템, 특성, 추천 파트너 정보를 확인하세요.` — 모든 포켓몬이 이름만 다르고 100% 동일. 검색엔진은 boilerplate description을 thin content로 인식하여 인덱싱 후순위 또는 미노출 처리하는 경향
2. **콘텐츠 깊이 본편 대비 적음**: `ChampionsDetailContainer` 152라인 (본편 detail은 폼별 라우팅 + 컨테이너 분리로 훨씬 풍부)
3. **OG 이미지 본편과 공유**: `getOgImageUrls`가 본편 이미지 폴더(default/form/mega/region/gigantamax) 사용 → SNS/검색 미리보기에서 본편 detail과 차별화 안 됨
4. **사이트맵 priority 미차별**: 챔피언스 상세 `priority: 0.7`로 본편 detail과 동일. 신규 페이지인데 크롤링 우선순위 가중치 없음
5. **인입 경로 한정**: 챔피언스 상세 페이지로의 진입은 `/champions/list` → 카드 클릭만 존재. 홈/타입 상성 등 메인 트래픽 페이지에서 챔피언스 상세로의 직접 링크 부재

### W4. 네이버 SEO 구조적 갭 — 점수 2/5

**근거 (지표)**: 네이버 한국 검색 점유율 62.86% 대비 포케코리아 네이버 유입 36%(모바일 30.1% + PC 5.9%). 구글 유입(40.5%)이 오히려 더 높은 역전 구조.

**근거 (네이버 서치어드바이저 실측 데이터)**:

- /detail/902 (대쓰여너): 노출 7,959 → 클릭 192 (CTR **2.4%**)
- /type-effectiveness: 노출 7,431 → 클릭 150 (CTR **2.0%**)
- /detail/908?shinyMode=shiny: 노출 1,990 → 클릭 47 (CTR **2.4%**)
- /detail/952/mega: 노출 2,192 → 클릭 41 (CTR **1.9%**)

핵심 페이지에서 노출은 충분하나 클릭이 평균 2~3% 수준. 평균 검색 순위 6.8 대비 비정상적으로 낮은 CTR이 네이버 SERP에서 메타 description 노출 품질이 부족함을 시사. 반면 /quiz/type-effectiveness는 CTR 22.2%로 동일 사이트 내에서도 페이지별 격차가 매우 크다.

**근거 (코드)**: naver-site-verification은 확인됐으나, 네이버 전용 메타 최적화(description 60~80자 이내 조정) 흔적은 `createMetadata` 헬퍼(`src/constants/seoMetaData.ts`)에서 미확인. 현재 description은 구글 기준으로 작성되어 있으며 네이버 표시 기준(약 80자 노출)에 맞는 별도 최적화가 없음.

### W5. DAU/MAU 4.2% — 리텐션 취약 — 점수 2/5

**근거 (지표)**: 정상 도구 사이트 DAU/MAU 10~20% 대비 4.2%는 재방문 구조가 사실상 없는 수준. D1/D7/D30 리텐션 미측정 상태.

**근거 (코드 추정)**: 재방문을 유도할 수 있는 "저장/즐겨찾기 기능", "대전 기록", "알림" 기능이 코드베이스에서 미확인. 퀴즈 페이지가 유일한 게이미피케이션 요소이나 매일 퀴즈는 홈에만 노출.

### W6. 챔피언스 데미지 계산기/팀빌더 미구현 — 점수 2/5

**근거 (코드)**: `/champions` 하위 구현 확인 결과 `/champions`, `/champions/list`, `/champions/tier` 3개 페이지만 존재. OP.GG가 이미 데미지 계산기, 팀빌더, 기술 919종 커버리지를 제공(MI 보고서)하는 반면 포케코리아는 챔피언스 전용 계산 도구 미구현 상태. `ChampionsHomeContainer`는 티어별 카드 나열에 그치며 도구적 가치가 낮음.

## 5. 기회 (Opportunities)

### O1. 챔피언스 모바일 출시(2026 여름) — 단기 최대 트래픽 급증

MI 보고서 발견 3과 연결. 모바일 배틀 게임 특성상 실전 중 검색 패턴 강화 → "챔피언스 타입 상성", "챔피언스 기술 목록", "챔피언스 메가진화 포켓몬" 검색 폭증 예상. 현재 코드베이스에 `/champions/list`(도감)와 `/champions/tier`(티어)는 구현돼 있어 최소 진입점은 존재. 챔피언스 개별 포켓몬 상세 페이지 선제 구축 시 해당 기간 장악 가능.

### O2. 본편(Z-A) + 챔피언스 통합 커버리지 차별화

MI 보고서 발견 1과 연결. OP.GG는 챔피언스만 커버하고 Z-A, 스칼렛/바이올렛 미커버. 포케코리아는 이미 1025마리 본편 도감 및 기술 800개 이상 보유(homeMetadata 기준). "Z-A 포켓덱스", "Z-A 타입 상성" 키워드에서 OP.GG와 경쟁 없는 독점 포지션 유지 가능.

### O3. 나무위키 불가 도구 키워드 블루오션

MI 보고서 발견 5와 연결. 나무위키가 정보 키워드 1~2위를 장악하나 인터랙티브 도구는 구조적으로 제공 불가. "포켓몬 상성 계산기", "포켓몬 데미지 계산기", "능력치 계산기" 키워드에서 외부 사이트에 기회 존재. `/type-effectiveness` 성공 사례가 이를 증명.

### O4. 네이버 블로그 보조 채널

## 6. 위협 (Threats)

### T1. OP.GG 챔피언스 서비스 확장 — 위협 수준 매우 높음

MI 보고서 섹션 4-2와 연결. 한국 51위(월 6,191만 방문) 도메인 권위와 "LoL = OP.GG" 브랜드 신뢰가 포켓몬으로 확장. 이미 타입 상성표, 데미지 계산기, 258종 스탯/기술 데이터를 Beta로 제공 중. 정식 출시 및 Beta 딱지 제거 시 "포켓몬 챔피언스 도구" 키워드 장악 가능성 높음. 현재 코드 점검 결과 포케코리아의 챔피언스 전용 계산 도구가 미구현(W6)이므로 차별화 없이는 피해 불가피.

### T2. 나무위키 정보 키워드 지속 장악

MI 보고서 섹션 4-1과 연결. 한국 내 7위 사이트(월간 수천만 방문)가 "포켓몬 [이름]", "포켓몬 레전드 Z-A" 등 정보형 키워드 1~2위 점령. 포케코리아가 정보 콘텐츠 방향으로 확장 시 직접 경쟁에서 도메인 권위 열세.

### T3. 구글 CTR 2.29% — 순위 정체 시 광고 수익 비례 정체

현재 AdSense 단일 수익 구조에서 MAU 성장이 없으면 수익 성장도 없음. 순위 6.8위 유지 시 CTR 개선 여지가 크지 않아 수익 증대 한계. AI Overview(구글 AI 답변 요약) 확대로 검색 클릭 자체가 감소하는 추세(추정)도 위협 요소.

### T4. 포켓몬 챔피언스 모바일 출시 선점 실패 위험

O1이 기회이기도 하나, 모바일 출시 전 OP.GG가 모바일 최적화를 완료하고 마케팅을 집중할 경우 검색 점유율을 선점당할 가능성. 챔피언스 상세 페이지 미구현(W3, W6) 상태에서 시간이 촉박.

## 8. 차별화 포인트 재정의

현재 `competitor-map.md`의 차별화 포인트("한국어 전용 도구", "모바일 최적화" 등)는 OP.GG 진입으로 일부 무력화됐다. 재정의 방향:

**기존 포지션**: "한국어로 포켓몬 도구를 제공하는 유일한 사이트"
→ OP.GG 한국어 서비스 런칭으로 "유일성" 훼손

**재정의 포지션 후보 3가지**:

1. **"본편 + 챔피언스 통합 원스톱 도구"**: OP.GG(챔피언스 전용), 나무위키(정보 전용)와 달리, 스칼렛/바이올렛 + Z-A + 챔피언스를 단일 사이트에서 커버하는 유일한 한국어 도구 사이트. 코드베이스 기반 실현 가능성 높음.

2. **"대전 유저를 위한 실전형 도구 특화"**: 타입 상성 계산기 → 데미지 계산기 → 챔피언스 팀빌더까지 대전 준비 전 과정을 커버. yakkun(일본어)의 한국어 대체재. 현재 `/type-effectiveness` 강점에서 자연스러운 확장.

3. **"신규 유저 진입장벽 낮춘 포켓몬 포털"**: 퀴즈, 오늘의 포켓몬, 챔피언스 티어 리스트 등 가볍게 탐색하는 3차 타겟(잠재 유저) 유입을 강화하고, 재방문율을 높이는 방향. 현재 DAU/MAU 4.2%를 10% 이상으로 끌어올리는 전략.

**추천 방향**: 1번(통합 커버리지) + 2번(대전 도구 깊이)의 조합. 단기(모바일 출시 전)는 챔피언스 도구 강화(2번), 중기는 본편 데이터 통합 강점 부각(1번).
