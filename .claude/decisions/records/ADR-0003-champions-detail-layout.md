# ADR-0003: 챔피언스 상세 페이지 데스크톱 레이아웃 구조

- **상태**: 승인
- **날짜**: 2026-04-19
- **담당**: Claude + 사용자

## 맥락

챔피언스 포켓몬 상세 페이지의 데스크톱 UI/UX 개선 작업 중, 정보 배치 방식에 대한 결정이 필요했다. 페이지에 표시해야 할 정보는 다음과 같다:

- 포켓몬 기본 정보 (이미지, 이름, 번호, 타입)
- 능력치 차트 (육각형 레이더 차트)
- 핵심 지표 (티어, 사용률, 승률)
- 메타 데이터 (인기 기술, 아이템, 특성, 추천 파트너)

정보량이 많아 한 화면에 모두 배치할 경우 밀도가 높아지고, 세로 스크롤로 나열할 경우 하단 정보 노출률이 저하될 우려가 있었다.

## 결정

**2단 가로형 구조 (좌측 고정 + 우측 스크롤)**를 채택한다.

```
┌────────────────────────┬────────────────────────────────────────┐
│   [좌측 - sticky]      │   [우측 - 스크롤]                       │
│                        │                                        │
│   포켓몬 이미지         │   핵심 지표 (티어/사용률/승률)           │
│   이름 / 번호 / 타입    │   인기 기술 (바 시각화)                 │
│   육각형 능력치 차트     │   인기 아이템                          │
│                        │   인기 특성                            │
│                        │   추천 파트너                          │
└────────────────────────┴────────────────────────────────────────┘
```

- 좌측: `position: sticky` + 고정 너비 (`w-96`)
- 우측: `flex-1` + `overflow-y: auto` + 최대 높이 제한

## 근거

### 1. 사용자 맥락 분석

챔피언스 메타 정보를 조회하는 사용자의 주요 목적:

- **게임 중 빠른 빌드 확인**: 핵심 정보를 즉시 파악해야 함
- **팀 빌딩 시 여러 포켓몬 비교**: 포켓몬 정체성(이미지/차트)을 고정하고 메타 정보만 탐색

→ 좌측 고정 구조가 "누구의 정보인지" 맥락을 유지하면서 메타 데이터 탐색에 집중할 수 있게 해준다.

### 2. 정보 계층 구조

| 우선순위 | 정보                                  | 배치                         |
| -------- | ------------------------------------- | ---------------------------- |
| 1순위    | 포켓몬 정체성 (이미지, 이름, 타입)    | 좌측 상단 (항상 노출)        |
| 2순위    | 핵심 지표 (티어, 사용률, 승률)        | 우측 상단 (스크롤 없이 노출) |
| 3순위    | 능력치 차트                           | 좌측 하단 (항상 노출)        |
| 4순위    | 메타 데이터 (기술/아이템/특성/파트너) | 우측 스크롤 영역             |

### 3. 스크롤 행동 연구

Nielsen Norman Group 연구에 따르면:

- 사용자는 스크롤을 꺼리지 않지만, **스크롤할 가치가 있다는 시각적 단서**가 필요
- 핵심 정보가 Above the fold에 있으면 하단 탐색 의지 증가

→ 핵심 지표를 우측 상단에 배치하여 "더 볼 가치 있는 정보가 있다"는 단서 제공

## 대안

| 대안                              | 장점                                 | 단점                                       | 불채택 사유                 |
| --------------------------------- | ------------------------------------ | ------------------------------------------ | --------------------------- |
| 3컬럼 가로 배치 (기존)            | 스크롤 없음, 한눈에 파악             | 메타 섹션 공간 부족, 바 시각화 적용 어려움 | 정보 밀도 과다, 확장성 부족 |
| 세로 섹션 스크롤                  | 각 섹션 충분한 공간, 모바일과 일관성 | 하단 정보 노출률 저하, PC에서 스크롤 많음  | 빠른 확인 목적에 부적합     |
| 탭 구조 (기술/아이템/특성/파트너) | 한 화면에 집중, 깔끔함               | 정보 비교 어려움, 클릭 필요                | 한눈에 전체 파악 불가       |

## 결과

### 예상 변화

1. 포켓몬 정체성(이미지/차트)이 스크롤해도 고정되어 맥락 유지
2. 메타 데이터 영역에 충분한 공간 확보 → 바 시각화 적용 가능
3. 핵심 지표(티어/사용률/승률)가 우측 상단에 강조 배치

### 구현 영향

- `ChampionsDetail.container.tsx` (데스크톱) 레이아웃 전면 수정
- `ChampionsMetaSection.component.tsx` 레이아웃 조정 필요
