경쟁 사이트의 특정 기능을 분석하여 구현 참고 자료를 정리해줘.

## 입력

- 분석 대상 기능: $ARGUMENTS (인자가 없으면 사용자에게 질문)

## 실행 절차

1. 분석할 기능명 확인
2. `references/competitor-sites.md`의 경쟁 사이트에서 해당 기능 조사
3. `references/output-format.md`의 형식에 따라 보고서 작성

## 조사 우선순위

1. 해당 기능에 특화된 사이트 (예: 데미지 계산기 → Smogon calc)
2. 종합 사이트의 해당 기능
3. 한국어/일본어 사이트의 해당 기능

## 주의사항

- 코드를 바로 작성하지 않는다. 분석 보고서만 작성한다.
- `/research` 스킬과의 차이: 이 스킬은 **경쟁 사이트의 특정 기능**에 집중, research는 **기술적 질문** 전반
- 검증되지 않은 정보는 명시적으로 "미검증"으로 표기한다.
