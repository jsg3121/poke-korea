경쟁 사이트의 특정 기능을 분석하여 구현 참고 자료를 정리해줘.

## 입력

- 분석 대상 기능: $ARGUMENTS (인자가 없으면 사용자에게 질문)

## 실행 절차

1. 분석할 기능명 확인
2. 주요 경쟁 사이트에서 해당 기능 조사
3. 각 사이트별 구현 방식 분석
4. poke-korea에 적용할 때의 차별화 포인트 도출
5. 보고서 형태로 정리

## 조사 대상 사이트

`references/competitor-sites.md` 참조

## 출력 형식

```
## 경쟁 분석 보고서: {기능명}

### 1. 기능 개요
{이 기능이 무엇이고 왜 사용자에게 필요한지}

### 2. 사이트별 구현 분석

#### {사이트명 1}
- **URL**: {해당 기능 페이지 URL}
- **핵심 기능**: {주요 기능 목록}
- **UI/UX**: {인터페이스 특징}
- **장점**: {잘 된 점}
- **단점**: {아쉬운 점}

#### {사이트명 2}
...

### 3. 공통 패턴
{대부분의 사이트가 공통으로 제공하는 기능/패턴}

### 4. 차별화 기회
{경쟁 사이트에 없거나 부족한 점 — poke-korea가 차별화할 수 있는 포인트}

### 5. 구현 권장사항

#### 필수 기능 (MVP)
- {기능 1}: {이유}
- {기능 2}: {이유}

#### 차별화 기능
- {기능 1}: {이유}
- {기능 2}: {이유}

#### 제외 권장
- {기능 1}: {이유 — 비용 대비 효과 낮음}

### 6. 한국어 시장 현황
{한국어로 이 기능을 제공하는 사이트 존재 여부 및 품질}

### 7. SEO 키워드 제안
{이 기능과 관련된 한국어 검색 키워드 목록}
```

## 조사 우선순위

1. 해당 기능에 특화된 사이트 (예: 데미지 계산기 → Smogon calc)
2. 종합 사이트의 해당 기능 (예: PokemonDB의 도구 섹션)
3. 한국어/일본어 사이트의 해당 기능
4. 커뮤니티 피드백 (Reddit, Twitter 등)

## 주의사항

- 코드를 바로 작성하지 않는다. 분석 보고서만 작성한다.
- `/research` 스킬과의 차이: 이 스킬은 **경쟁 사이트의 특정 기능**에 집중, research는 **기술적 질문** 전반
- 검증되지 않은 정보는 명시적으로 "미검증"으로 표기한다.
- 사이트 접근이 불가능한 경우 해당 사실을 명시한다.
