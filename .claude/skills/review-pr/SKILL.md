PR에 달린 코드 리뷰 코멘트를 분석하고 의견을 제시해줘.

## 입력

- PR 번호: $ARGUMENTS (인자가 없으면 사용자에게 PR 번호를 질문)

## 실행 절차

1. PR 기본 정보 확인 (`gh pr view {PR번호}`)
2. PR 리뷰/인라인/일반 코멘트 조회 (`gh api`)
3. 리뷰 코멘트가 참조하는 코드 파일을 직접 읽어서 현재 상태 확인
4. 각 코멘트를 **동의 / 검토 필요 / 반박** 으로 분류하여 분석

## 주의사항

- 리뷰 코멘트가 참조하는 코드를 반드시 직접 읽고 맥락을 파악한 뒤 의견을 제시한다.
- 프로젝트 CLAUDE.md 규칙과 충돌하는 경우, 프로젝트 규칙을 우선한다.
- 코드를 바로 수정하지 않는다. 수정 방안만 제안한다.
- `gh` CLI 인증이 안 되어 있으면 `gh auth login`을 안내한다.
