PR에 달린 코드 리뷰 코멘트를 분석하고 의견을 제시해줘.

## 입력

- PR 번호: $ARGUMENTS (인자가 없으면 사용자에게 PR 번호를 질문)

## 실행 절차

1. PR 기본 정보 확인 (`gh pr view {PR번호}`)
2. PR 리뷰 코멘트 조회 (`gh api repos/{owner}/{repo}/pulls/{PR번호}/reviews`)
3. PR 인라인 코드 리뷰 코멘트 조회 (`gh api repos/{owner}/{repo}/pulls/{PR번호}/comments`)
4. PR 일반 코멘트 조회 (`gh api repos/{owner}/{repo}/issues/{PR번호}/comments`)
5. 리뷰 코멘트가 참조하는 코드 파일을 직접 읽어서 현재 상태를 확인
6. 각 코멘트에 대해 분석 및 의견 제시

## 출력 형식

각 리뷰 코멘트에 대해 아래 형식으로 응답한다:

### 코멘트별 분석

각 코멘트를 아래 세 가지 중 하나로 **분류**한다:

- **동의** — 리뷰어 의견이 타당하며 수정이 필요한 경우
- **검토 필요** — 일부 동의하지만 방향에 대해 사용자와 논의가 필요한 경우
- **반박** — 리뷰어 의견이 잘못되었거나 프로젝트 규칙/버전과 맞지 않는 경우

```
📍 파일: {파일 경로}:{라인 번호}
💬 리뷰어: {리뷰어 이름}
📝 코멘트: {코멘트 내용 요약}
🏷️ 분류: {동의 | 검토 필요 | 반박}

🔍 분석:
- 분류 근거
- 현재 코드의 맥락 설명
- 수정이 필요한 경우 구체적인 수정 방안 제안
```

### 최종 요약

- 전체 리뷰 코멘트 수
- 동의 (수정 필요) 항목 수
- 반박 (무시해도 됨) 항목 수
- 검토 필요 항목 수

## 주의사항

- 리뷰 코멘트가 참조하는 코드를 반드시 직접 읽고 맥락을 파악한 뒤 의견을 제시한다.
- 리뷰어의 의견이 맞더라도 프로젝트의 CLAUDE.md 규칙과 충돌하는 경우, 프로젝트 규칙을 우선한다.
- 코드를 바로 수정하지 않는다. 수정 방안만 제안하고, 사용자가 요청할 때 수정한다.
- 반박 코멘트를 작성하는 경우 반드시 해당 코멘트의 리뷰어를 태깅하여 코멘트를 작성한다
- `gh` CLI 인증이 안 되어 있으면 사용자에게 `gh auth login`을 안내한다.
