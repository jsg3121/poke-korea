외부 정보를 조사하여 보고서 형태로 정리해줘.

## 트리거

사용자가 기술적 질문, 외부 라이브러리 최신 정보, 마이그레이션/업그레이드 계획을 요청할 때 자동 트리거

## 입력

- 조사 주제: $ARGUMENTS

## 실행 절차

1. 주제 파악 및 조사 범위 설정
2. **공식 문서를 최우선으로** 조사
3. GitHub 이슈/릴리즈 노트 확인 (해당 시)
4. 커뮤니티 사례 참고 (공식 문서 부족 시)
5. 핵심 요약 → 상세 내용 → 프로젝트 적용 고려사항 → 참고 자료 형태로 정리

## 주의사항

- 검증되지 않은 정보는 명시적으로 "미검증"으로 표기한다.
- 현재 프로젝트의 기술 스택 버전과의 호환성을 항상 확인한다.
- 코드를 바로 작성하지 않고 조사 결과만 보고한다.
