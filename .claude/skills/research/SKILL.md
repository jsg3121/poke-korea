외부 정보를 조사하여 보고서 형태로 정리해줘.

## 트리거

이 스킬은 다음 상황에서 자동으로 트리거됩니다:
- 사용자가 기술적 질문을 할 때
- 외부 라이브러리/프레임워크의 최신 정보가 필요할 때
- 마이그레이션이나 업그레이드 계획 시

## 입력

- 조사 주제: $ARGUMENTS

## 실행 절차

1. 주제 파악 및 조사 범위 설정
2. **공식 문서를 최우선으로** 조사
3. GitHub 이슈/릴리즈 노트 확인 (해당 시)
4. 커뮤니티 사례 참고 (공식 문서 부족 시)
5. 보고서 형태로 정리

## 조사 우선순위

1. 공식 문서 (최우선)
2. GitHub 릴리즈 노트 / 마이그레이션 가이드
3. RFC / 설계 문서
4. 공신력 있는 블로그 (팀 블로그, 메인테이너 블로그)
5. Stack Overflow / GitHub Discussions

## 출력 형식

```
## 조사 보고서: {주제}

### 핵심 요약
{1~3문장으로 핵심 결론}

### 상세 내용
{조사 결과를 구조화하여 정리}

### 프로젝트 적용 시 고려사항
{poke-korea 프로젝트에 적용할 때 주의할 점}

### 참고 자료
- {출처 1}: {URL 또는 문서명}
- {출처 2}: {URL 또는 문서명}
```

## 주의사항

- 검증되지 않은 정보는 명시적으로 "미검증"으로 표기한다.
- 현재 프로젝트의 기술 스택 버전과의 호환성을 항상 확인한다.
- 코드를 바로 작성하지 않고 조사 결과만 보고한다.
