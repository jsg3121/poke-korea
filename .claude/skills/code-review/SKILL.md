현재 브랜치의 변경사항에 대해 코드 리뷰를 수행해줘.

## 입력

- 대상: $ARGUMENTS (인자가 없으면 현재 브랜치의 전체 변경사항)

## 실행 절차

1. 현재 브랜치와 base 브랜치 간 변경사항 확인 (`git diff {base}...HEAD`)
2. 변경된 파일 목록 확인 (`git diff {base}...HEAD --name-only`)
3. **서브에이전트를 사용하여** 사전 맥락 없이 순수하게 코드만 보고 리뷰 수행
4. Critical / Warning / Suggestion으로 분류하여 보고

## 리뷰 관점

- **필수**: 타입 안전성, 잠재적 버그, 프로젝트 컨벤션 위반
- **권장**: 성능, 접근성, SEO (해당 시)

## 주의사항

- 코드를 바로 수정하지 않는다. 리뷰 결과만 보고한다.
- 프로젝트 규칙(`.claude/conventions/`)을 기준으로 판단한다.
