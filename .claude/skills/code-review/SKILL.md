현재 브랜치의 변경사항에 대해 코드 리뷰를 수행해줘.

## 입력

- 대상: $ARGUMENTS (인자가 없으면 현재 브랜치의 전체 변경사항)

## 실행 절차

1. 현재 브랜치와 base 브랜치 간 변경사항 확인 (`git diff {base}...HEAD`)
2. 변경된 파일 목록 확인 (`git diff {base}...HEAD --name-only`)
3. **서브에이전트를 사용하여** 사전 맥락 없이 순수하게 코드만 보고 리뷰 수행
4. 리뷰 결과를 종합하여 보고

## 리뷰 관점

### 필수 검토

- **타입 안전성**: `any` 사용, 타입 단언(`as`) 남용, 누락된 타입
- **잠재적 버그**: null/undefined 처리, 비동기 에러 처리, 무한 루프/렌더링
- **프로젝트 컨벤션 위반**: `.claude/conventions/guides/` 규칙 대비

### 권장 검토

- **성능**: 불필요한 리렌더링, 메모이제이션 누락, 큰 번들
- **접근성**: aria 속성, 시맨틱 마크업
- **SEO**: 메타데이터 누락 (해당 시)

## 출력 형식

```
## 코드 리뷰 결과

### 🔴 Critical (반드시 수정)
- {파일}:{라인} — {설명}

### 🟡 Warning (수정 권장)
- {파일}:{라인} — {설명}

### 🟢 Suggestion (선택적 개선)
- {파일}:{라인} — {설명}

## 요약
- Critical: {N}건
- Warning: {N}건
- Suggestion: {N}건
```

## 주의사항

- 코드를 바로 수정하지 않는다. 리뷰 결과만 보고한다.
- 프로젝트 규칙(`.claude/conventions/`)을 기준으로 판단한다.
