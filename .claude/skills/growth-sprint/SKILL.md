성장 실험을 설계하고 Growth Sprint 계획을 수립해줘.

## 트리거

이 스킬은 다음 상황에서 자동으로 트리거됩니다:
- 사용자가 "성장 실험", "A/B 테스트", "신규 채널", "스프린트" 등을 요청할 때
- 특정 지표를 개선하기 위한 실험 설계가 필요할 때

## 입력

- 실험 주제 또는 개선 대상 지표: $ARGUMENTS (인자가 없으면 사용자에게 질문)

## 실행 순서

1. 최근 weekly 분석 결과 로드 (`.claude/outputs/weekly/` 최신 파일)
2. 이전 실험 결과 확인 (`.claude/outputs/experiments/`)
3. growth-strategist 에이전트 실행하여 실험 설계
4. 실험 설계서를 `.claude/outputs/experiments/[실험명].md`에 저장

## 실험 설계 필수 항목

`references/experiment-template.md` 참조

## ICE 점수 기준

| 점수 | Impact | Confidence | Ease |
|------|--------|------------|------|
| 8~10 | North Star 지표에 직접 영향 | 유사 실험 선례 있거나 데이터 근거 충분 | 1주 이내 실행 가능 |
| 5~7 | 퍼널 중간 지표에 영향 | 논리적 근거는 있으나 직접 데이터 부족 | 2~4주 소요 |
| 1~4 | 간접적 효과만 기대 | 가설 수준, 검증 데이터 없음 | 1개월 이상 소요 |

## 출력 형식

```
## Growth Sprint: {실험 주제}

### 배경
{왜 이 실험이 필요한지 — weekly review 결과 기반}

### 실험 설계

#### 실험 1: {이름}
- 가설: [이것을 하면] [이 지표가] [얼마나] 변할 것이다
- ICE 점수: I({n}) × C({n}) × E({n}) = {total}
- 측정 방법: {구체적 측정 도구 및 지표}
- 성공 기준: {수치 기반 판단 기준}
- 필요 리소스: {구현 작업, 예상 소요 시간}
- 실행 기간: {시작일 ~ 종료일}

### 우선순위 (ICE 점수 기반)
| 순위 | 실험명 | ICE | 실행 기간 |
|------|--------|-----|-----------|

### 구현 시 참고사항
{poke-korea 프로젝트 구조에서 수정이 필요한 영역}
```

## 주의사항

- 코드를 바로 작성하지 않는다. 실험 설계서만 작성한다.
- 모든 가설은 측정 가능한 형태로 기술한다.
- 이전 실험 결과가 있다면 반드시 참조하여 중복 실험을 방지한다.
- 구현이 필요한 실험은 관련 에이전트(ui-publisher, seo-specialist 등)와의 협업 계획을 포함한다.
