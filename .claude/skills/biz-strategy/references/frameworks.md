# 비즈니스 분석 프레임워크

에이전트가 분석 시 참조하는 프레임워크 가이드이다.

---

## 거시환경 분석: PESTLE

외부 환경 요인을 6가지 차원으로 분석한다.

| 요인 | 분석 대상 | ConfigDeck 예시 |
|------|----------|----------------|
| **P**olitical (정치) | 규제, 정책, 무역 | 오픈소스 라이선스 정책 변화 |
| **E**conomic (경제) | 경기, 환율, 투자 | 개발자 채용 시장, SaaS 지출 트렌드 |
| **S**ocial (사회) | 인구, 문화, 트렌드 | 원격 근무 확대, 사이드 프로젝트 문화 |
| **T**echnological (기술) | 기술 발전, 혁신 | AI 코드 생성, 새로운 빌드 도구 등장 |
| **L**egal (법률) | 법규, 지적재산 | 설정 파일의 저작권, CC0 라이선스 |
| **E**nvironmental (환경) | 환경 요인 | (해당 사항 적음) |

## 산업구조 분석: Porter's 5 Forces

산업의 경쟁 강도와 수익성을 5가지 힘으로 분석한다.

| 힘 | 질문 | 분석 포인트 |
|----|------|------------|
| 기존 경쟁자 | 현재 얼마나 많은 유사 서비스가 있는가? | gitignore.io, PureDevTools, 각종 config generator |
| 신규 진입자 | 새로운 경쟁자가 진입하기 쉬운가? | 진입 장벽 낮음 (정적 사이트로 구현 가능) |
| 대체재 | 다른 방법으로 같은 문제를 해결할 수 있는가? | CLI 도구, 보일러플레이트, AI 코드 생성 |
| 공급자 교섭력 | 핵심 리소스 공급자의 영향력은? | (해당 사항 적음 — 오픈소스 기반) |
| 구매자 교섭력 | 사용자의 전환 비용은 얼마나 낮은가? | 매우 낮음 (무료 서비스, 전환 비용 0) |

## 내부 역량 분석: SWOT

| | 긍정적 | 부정적 |
|---|--------|--------|
| **내부** | Strengths (강점) | Weaknesses (약점) |
| **외부** | Opportunities (기회) | Threats (위협) |

### SWOT 작성 원칙
- 각 항목에 최소 3개, 최대 5개
- 항목마다 "왜 그런가?" 근거 1줄 필수
- SO/WO/ST/WT 전략 매트릭스 도출 권장

## 포지셔닝 분석: Value Proposition Canvas

| 고객 측 | 제품 측 |
|---------|---------|
| Customer Jobs (해결할 과제) | Products & Services |
| Pains (고통점) | Pain Relievers |
| Gains (기대 이득) | Gain Creators |

## 경쟁 우위 분석: Jobs-to-be-Done

사용자가 "고용(hire)"하는 이유를 기능적/감정적/사회적 차원으로 분석한다.

| 차원 | 질문 | 예시 |
|------|------|------|
| 기능적 | 어떤 작업을 완료하려 하는가? | 프로젝트 설정 파일을 빠르게 생성 |
| 감정적 | 어떤 기분을 느끼고 싶은가? | 설정 누락 없이 확신을 갖고 시작 |
| 사회적 | 어떻게 보이고 싶은가? | 최신 모범 사례를 따르는 개발자 |

## 시장 규모 추정: TAM/SAM/SOM

| 단계 | 정의 | 추정 방법 |
|------|------|----------|
| TAM (Total Addressable Market) | 전체 잠재 시장 | 전 세계 개발자 수 × 설정 파일 관련 시간 비용 |
| SAM (Serviceable Available Market) | 접근 가능 시장 | 웹 기반 config 생성기를 사용할 개발자 |
| SOM (Serviceable Obtainable Market) | 실제 획득 가능 시장 | 다국어 지원 + SEO 유입으로 확보 가능한 사용자 |
