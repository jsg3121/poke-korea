PR 리뷰 코멘트에 대한 답글을 GitHub에 작성해줘.

## 입력

- PR 번호: $ARGUMENTS (인자가 없으면 사용자에게 PR 번호를 질문)

## 선행 조건

- **반드시 같은 대화에서 `/review-pr {PR번호}`가 먼저 실행되어 있어야 한다.**
- `/review-pr`이 실행되지 않았다면, 먼저 실행하도록 안내하고 중단한다.

## 코멘트 작성 규칙

- **동의 항목**: 코멘트 남기지 않음. 코드 수정만 진행.
- **검토 필요 항목**: 사용자에게 방향 확인 후, 리뷰어와 다른 방향일 때만 코멘트.
- **반박 항목**: 근거(공식 문서, 버전, 프로젝트 규칙 등)를 포함하여 코멘트.

## 실행 절차

1. `/review-pr` 분석 결과에서 코멘트 대상 추출
2. PR 인라인 코멘트의 `id` 확인 및 기존 답글 여부 확인
3. 답글이 없는 코멘트에 대해 작성 내용을 사용자에게 보여주고 확인
4. 확인 후 `gh api`로 답글 작성 (코멘트 본문 앞에 `@{리뷰어}` 멘션)

## 주의사항

- 정중하고 건설적인 톤으로 작성한다.
- 이미 답글이 달린 코멘트에는 중복 코멘트를 남기지 않는다.
- `gh` CLI 인증이 안 되어 있으면 `gh auth login`을 안내한다.
