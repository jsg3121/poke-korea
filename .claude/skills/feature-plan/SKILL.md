신규 기능의 기획서를 작성하여 구현 전 요구사항과 설계를 정리해줘.

## 입력

- 기능명: $ARGUMENTS (인자가 없으면 사용자에게 질문)

## 실행 절차

1. 기능명을 기반으로 프로젝트 현황 파악
   - 관련 기존 코드 확인 (라우트, Context, GraphQL 쿼리 등)
   - 유사한 기존 기능이 있다면 구현 패턴 참고
2. 경쟁 사이트에서 해당 기능이 어떻게 구현되어 있는지 간략 조사
3. `references/output-format.md`의 형식에 따라 기획서 작성
4. 사용자에게 기획서 검토 요청

## 기획서 포함 항목

기능 개요, 사용자 시나리오, 데이터 요구사항, 페이지 구조, 상태 관리, 계산 로직, SEO, 구현 단계, 에이전트 협업 계획

## 주의사항

- 코드를 바로 작성하지 않는다. 기획서만 작성한다.
- 사용자가 기획서를 승인한 후에 구현을 시작한다.
- 기존 프로젝트 패턴(desktop/mobile 분리, Context 패턴 등)을 반드시 반영한다.
