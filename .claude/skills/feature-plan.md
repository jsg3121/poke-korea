신규 기능의 기획서를 작성하여 구현 전 요구사항과 설계를 정리해줘.

## 입력

- 기능명: $ARGUMENTS (인자가 없으면 사용자에게 질문)

## 실행 절차

1. 기능명을 기반으로 프로젝트 현황 파악
   - 관련 기존 코드 확인 (라우트, Context, GraphQL 쿼리 등)
   - 유사한 기존 기능이 있다면 구현 패턴 참고
2. 경쟁 사이트에서 해당 기능이 어떻게 구현되어 있는지 간략 조사
3. 기획서 작성 (아래 출력 형식 참조)
4. 사용자에게 기획서 검토 요청

## 기획서에 포함할 항목

| 항목 | 설명 |
|------|------|
| 기능 개요 | 무엇을 만들고 왜 필요한지 |
| 사용자 시나리오 | 사용자가 이 기능을 어떻게 사용하는지 단계별 흐름 |
| 데이터 요구사항 | 필요한 GraphQL 쿼리/데이터, 새 쿼리 필요 여부 |
| 페이지 구조 | 라우트 경로, 컴포넌트 계층 |
| 상태 관리 | Context 설계, localStorage 활용 여부 |
| 계산 로직 | 핵심 비즈니스 로직 (해당 시) |
| SEO | 메타태그, JSON-LD 스키마, sitemap 반영 |
| 구현 단계 | 작업을 나누어 진행할 단계 목록 |

## 출력 형식

```
## 기능 기획서: {기능명}

### 1. 기능 개요
{무엇을 만들고, 왜 필요한지}

### 2. 사용자 시나리오
1. {사용자 행동 1}
2. {사용자 행동 2}
3. ...

### 3. 데이터 요구사항
- 기존 쿼리 활용: {쿼리명 목록}
- 새 쿼리 필요: {필요 여부 및 내용}

### 4. 페이지 구조
- 라우트: `/경로`
- 컴포넌트 계층:
  ```
  page.tsx → views → container → components
  ```

### 5. 상태 관리
- Context: {Context명 및 관리할 상태}
- localStorage: {저장 항목 및 키 구조} (해당 시)

### 6. 계산 로직
{핵심 공식, 알고리즘, 참조 문서} (해당 시)

### 7. SEO
- title: {제안}
- description: {제안}
- JSON-LD: {스키마 유형}

### 8. 구현 단계
- [ ] 단계 1: {내용}
- [ ] 단계 2: {내용}
- [ ] ...

### 9. 에이전트 협업 계획
{어떤 에이전트가 어떤 단계를 담당하는지}
```

## 주의사항

- 코드를 바로 작성하지 않는다. 기획서만 작성한다.
- 사용자가 기획서를 승인한 후에 구현을 시작한다.
- 기존 프로젝트 패턴(desktop/mobile 분리, Context 패턴 등)을 반드시 반영한다.
- 구현 단계는 CLAUDE.md의 "CRITICAL: 복잡한 기능 구현 시 각 작업 단계를 나누어 진행" 규칙을 따른다.
