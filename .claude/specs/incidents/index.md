# Incidents

운영 환경에서 발생한 이슈의 디버깅 과정과 해결 히스토리를 보존하는 폴더입니다.

## 목적

- 이슈 발생 시 어떤 가설을 세웠고 어떻게 검증했는지 사고 과정을 기록
- 막다른 길로 갔던 가설도 "왜 틀렸는지" 와 함께 기록
- 향후 비슷한 이슈 발생 시 빠르게 좁힐 수 있도록 참고 자료 제공
- 자기 비판적 회고를 통해 진단/운영/방법론 측면의 학습 포인트 도출

## 무엇을 기록하는가

- **이슈 개요**: 무엇이 발생했고 어떤 증상이었는지
- **초기 의심 가설**: 어떤 시나리오를 의심했고 왜 그렇게 추측했는지
- **검증 과정**: 가설별로 어떻게 검증했고 어떻게 반박/확정되었는지
- **막다른 길**: 틀린 추론도 명시적으로 기록 (왜 틀렸는지 포함)
- **핵심 개념 정리**: 분석 과정에서 정리된 시스템 동작 원리
- **실제 검증 데이터**: curl 결과, 매트릭스, 코드 인용 등
- **근본 원인**: 직접 원인과 간접 원인 (운영 vs 코드 분포)
- **해결 방안**: 선택된 방안과 그 이유 (다른 옵션과의 비교)
- **잔존 위험 및 후속 작업**: 1차 대응으로 못 막은 부분
- **학습 포인트**: 진단/운영/방법론 측면의 회고
- **참고 자료**: 공식 문서, RFC, 관련 PR/changelog 링크

## 기록 원칙

1. **근거 기반**: 추측은 명시적으로 표시하고, 결론은 공식 문서/검증 데이터로 뒷받침
2. **사고 과정 보존**: 결론만 적지 않고 어떻게 그 결론에 도달했는지 서사 기록
3. **막다른 길 포함**: 틀린 가설도 가치 있는 정보. 다음 번에 같은 함정을 피하기 위해
4. **자기 비판 가감 없이**: 운영 실수, 진단 미스, 미해결 숙제 방치 등 솔직하게 기록
5. **버전 무관**: 코드 변경이 아닌 운영 이슈 보존이 목적이므로 changelog 와 별개로 관리

## 파일 네이밍 규칙

