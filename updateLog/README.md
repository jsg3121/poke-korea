# 포케코리아 업데이트 로그

이 디렉토리는 포케코리아(poke-korea.com)의 모든 버전별 변경사항을 기록합니다.

---

## 📋 로그 관리 규칙

### 1. 파일 네이밍
```
{버전}.md
예: 1.26.0.md, 1.27.0.md
```

### 2. 버전 번호 규칙
[Semantic Versioning](https://semver.org/lang/ko/)을 따릅니다:
- **Major (X.0.0)**: 대규모 변경, Breaking Changes
- **Minor (1.X.0)**: 새로운 기능 추가, 하위 호환성 유지
- **Patch (1.26.X)**: 버그 수정, 소규모 개선

### 3. 브랜치 관리
```
feature/{버전} 형식으로 브랜치 생성
예: feature/1.26.0

해당 브랜치가 main에 머지되기 전까지의 모든 변경사항을 기록
```

### 4. 로그 작성 시점
- 브랜치 작업 시작 시: 파일 생성
- 주요 기능 추가/수정 시: 실시간 업데이트
- main 머지 전: 최종 검토 및 완성
- main 머지 후: 릴리즈 날짜, PR 링크 추가

## 📝 로그 템플릿

각 버전 로그는 다음 섹션을 포함해야 합니다:

### 필수 섹션
- **주요 변경사항**: 이번 버전의 핵심 내용 요약
- **기능 개선**: 새로 추가된 기능
- **버그 수정**: 수정된 버그 목록
- **커밋 히스토리**: 모든 커밋 메시지

### 선택 섹션 (해당 시)
- **SEO 개선**: SEO 관련 작업
- **코드 리팩토링**: 대규모 리팩토링
- **성능 개선**: 성능 최적화 작업
- **디자인 변경**: UI/UX 변경사항
- **기술적 개선사항**: 인프라, 빌드 설정 등

## 🤝 기여 가이드

### 로그 작성 시 주의사항
1. **명확한 제목**: 각 섹션의 제목은 명확하고 구체적으로
2. **코드 예시**: 변경사항은 Before/After 코드로 설명
3. **파일 링크**: 변경된 파일은 상대 경로로 링크
4. **통계 제공**: 정량적 지표 포함 (줄 수, 파일 수 등)
