## 🎯 주요 변경사항

### 1. 홈페이지 구조 개편

- 기존 루트 경로(`/`)의 포켓몬 리스트를 `/list` 경로로 이동
- 새로운 홈페이지 디자인 준비 (향후 다양한 컨텐츠 진입점 제공)
- 검색 순위 유지를 위한 308 Permanent Redirect 구현

### 2. 대규모 코드 리팩토링

- **중복 코드 200줄 제거** (8개 컨테이너 + 4개 Context/Hook)
- **3개 공통 모듈 생성**으로 재사용성 향상
- 파일 복잡도 평균 21.6% 감소

### 3. SEO 최적화

- Sitemap에 32개 새 URL 추가
- Breadcrumb JSON-LD 구조화 데이터 추가
- ItemList Schema 구현 (일일 랜덤 10개 포켓몬)
- 필터별 동적 메타데이터 생성

## 🔍 SEO 개선

### Sitemap 확장

새로 추가된 URL (총 32개):

- `/list` - 포켓몬 도감 메인 (priority: 0.9)
- 타입 필터: 18개 URL (`/list?type=NORMAL`, `/list?type=FIRE` 등)
- 세대 필터: 9개 URL (`/list?generation=1` ~ `/list?generation=9`)
- Boolean 필터: 4개 URL
  - `/list?isMega=true` - 메가진화 포켓몬
  - `/list?isRegion=true` - 지역 폼 포켓몬
  - `/list?isEvolution=true` - 진화 포켓몬
  - `/list?isEvolution=false` - 미진화 포켓몬

**파일:** [src/app/sitemap.ts](../src/app/sitemap.ts)

### 구조화 데이터 (Schema.org)

#### 1. Breadcrumb JSON-LD

리스트 페이지에 빵 부스러기 경로 추가:

```json
{
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "position": 1, "name": "홈", "item": "https://poke-korea.com" },
    {
      "position": 2,
      "name": "포켓몬 도감",
      "item": "https://poke-korea.com/list"
    }
  ]
}
```

#### 2. ItemList Schema

매일 랜덤으로 선정된 10개 포켓몬을 구조화 데이터로 제공:

```json
{
  "@type": "ItemList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "item": {
        "@type": "Thing",
        "name": "피카츄",
        "url": "https://poke-korea.com/detail/25"
      }
    }
    // ... 총 10개
  ]
}
```

**알고리즘:** LCG (Linear Congruential Generator)를 사용한 일일 랜덤 선정
**파일:** [src/module/list.module.ts](../src/module/list.module.ts)

### 동적 메타데이터

필터별 최적화된 제목과 설명 생성:

// Before
const listRef = useRef<HTMLDivElement>(null)

      loadMore()
    }

}

useEffect(() => {
  const observer = new IntersectionObserver(observerCallback, {
    root: null,
    rootMargin: '0px 0px 380px 0px',
    threshold: 0,

    observer.observe(listRef.current)

  loadMore,
  rootMargin: '0px 0px 380px 0px',
  dependencies: [pokemonList],

### 2. GraphQL 페이지네이션 유틸 통합

#### 새로 생성된 파일

- **[src/module/graphqlPagination.module.ts](../src/module/graphqlPagination.module.ts)** (+94줄)

#### 제공하는 유틸 함수

1. **`mergePagedResults<T>`** - Relay 커서 페이지네이션 결과 병합
2. **`extractNodesFromEdges<T>`** - edges 배열에서 node 추출
3. **`createPaginationVars`** - 페이지네이션 변수 생성 헬퍼

#### 영향받은 파일 (4개)

1. [src/context/List.context.tsx](../src/context/List.context.tsx) - `getPokemonList`
2. [src/context/Moves.context.tsx](../src/context/Moves.context.tsx) - `getPokemonSkillList`
3. [src/hook/useAbilityList.ts](../src/hook/useAbilityList.ts) - `getAbilityListPaginated`
4. [src/hook/usePokemonByAbility.ts](../src/hook/usePokemonByAbility.ts) - `getPokemonByAbility`

#### 개선 효과

```diff
- 각 파일마다 20줄의 페이지네이션 로직 중복
+ 2줄의 유틸 함수 호출로 대체

Before: 4개 파일 × 20줄 = 80줄
After:  1개 모듈 94줄 + 4개 × 2줄 = 102줄
순 증가: 22줄 (하지만 타입 안전성과 재사용성 크게 향상)
```

#### 사용 예시

```typescript
// Before
import { produce } from 'immer'
import { PokemonEdge } from '~/graphql/typeGenerated'

await fetchMore({
  updateQuery: (previousQueryResult, { fetchMoreResult }) => {
    if (!fetchMoreResult) return previousQueryResult

    return produce(previousQueryResult, (draft) => {
      draft.getPokemonList.edges = [
        ...previousQueryResult.getPokemonList.edges,
        ...fetchMoreResult.getPokemonList.edges,
      ]
      draft.getPokemonList.pageInfo = {
        ...previousQueryResult.getPokemonList.pageInfo,
        endCursor: fetchMoreResult.getPokemonList.pageInfo.endCursor,
        hasNextPage: fetchMoreResult.getPokemonList.pageInfo.hasNextPage,
      }
    })
  },
})

const pokemonList =
  data?.getPokemonList?.edges?.map((edge: PokemonEdge) => edge.node) ||
  initialList

// After
import {
  mergePagedResults,
  extractNodesFromEdges,
} from '~/module/graphqlPagination.module'

await fetchMore({
  updateQuery: (prev, { fetchMoreResult }) =>
    mergePagedResults('getPokemonList', prev, fetchMoreResult),
})

const pokemonList = extractNodesFromEdges(
  data?.getPokemonList?.edges,
  initialList,
)
```

## 🐛 버그 수정

### 1. 308 Redirect 구현

**문제:** 기존 URL(`/?type=FIRE`)에서 검색 순위 유지 필요
**해결:** 쿼리 파라미터 있을 때만 `/list`로 308 Permanent Redirect

```typescript
// src/app/page.tsx
if (searchParams && Object.keys(searchParams).length > 0) {
  const queryString = new URLSearchParams(searchParams).toString()
  permanentRedirect(`/list?${queryString}`)
}
```

### 2. 타입 안전성 개선

**문제:** `hasNextPage`가 `undefined`일 수 있는데 `boolean`으로만 타입 지정
**해결:** `boolean | undefined` 타입으로 수정

```typescript
// src/hook/useInfiniteScroll.ts
interface UseInfiniteScrollProps {
  hasNextPage: boolean | undefined // ✅ undefined 허용
  // ...
}
```
