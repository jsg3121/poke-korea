### Phase 4: Portal 및 팝업 컴포넌트

React Portal API를 사용한 재사용 가능한 모달 시스템:

#### 1. Portal 컴포넌트

```tsx
// src/components/Portal.component.tsx
const Portal = ({ children, containerId = 'portal-root' }) => {
  const [mounted, setMounted] = useState(false)
  const [portalElement, setPortalElement] = useState<HTMLElement | null>(null)

  useEffect(() => {
    setMounted(true)
    let element = document.getElementById(containerId)

    // portal-root가 없으면 생성
    if (!element) {
      element = document.createElement('div')
      element.id = containerId
      document.body.appendChild(element)
    }

    setPortalElement(element)

    // cleanup: 자식 노드가 없으면 제거
    return () => {
      if (element && element.childNodes.length === 0) {
        element.remove()
      }
    }
  }, [containerId])

  if (!mounted || !portalElement) return null
  return createPortal(children, portalElement)
}
```

**특징:**

- SSR 호환 (mounted 상태 체크)
- 자동 cleanup (메모리 누수 방지)
- 재사용 가능한 containerId

**파일:** [src/components/Portal.component.tsx](../src/components/Portal.component.tsx)

#### 2. QuizResultPopup 컴포넌트

```tsx
// src/components/QuizResultPopup.component.tsx
<Portal>
  <div
    role="dialog"
    aria-modal="true"
    aria-labelledby="quiz-result-title"
    aria-describedby="quiz-result-description"
  >
    {/* Backdrop with blur */}
    <div className="bg-black/50 backdrop-blur-sm" />

    {/* Result Icon with animation */}
    {isCorrect ? (
      <div className="animate-bounce">✅</div>
    ) : (
      <div className="animate-shake">❌</div>
    )}

    {/* Result Text */}

  </div>
```

**애니메이션:**

- `fadeIn` - 백드롭 페이드 인 (0.2초)
- `scaleIn` - 모달 스케일 인 (0.3초)
- `bounce` - 정답 시 튀는 효과
- `shake` - 오답 시 흔들림 효과

**접근성:**

- `role="dialog"`, `aria-modal="true"` - 스크린 리더 지원
- `aria-labelledby`, `aria-describedby` - 제목/설명 연결
- `body` 스크롤 잠금 (팝업 열릴 때)
- ESC 키로 닫기 (키보드 네비게이션)

#### 2. ItemList Schema

매일 랜덤으로 선정된 10개 포켓몬을 구조화 데이터로 제공:

```json
{
  "@type": "ItemList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "item": {
```

### 2. GraphQL 페이지네이션 유틸 통합

#### 새로 생성된 파일

#### 제공하는 유틸 함수

1. **`mergePagedResults<T>`** - Relay 커서 페이지네이션 결과 병합
2. **`extractNodesFromEdges<T>`** - edges 배열에서 node 추출
3. **`createPaginationVars`** - 페이지네이션 변수 생성 헬퍼

#### 영향받은 파일 (4개)

1. [src/context/List.context.tsx](../src/context/List.context.tsx) - `getPokemonList`
2. [src/context/Moves.context.tsx](../src/context/Moves.context.tsx) - `getPokemonSkillList`
3. [src/hook/useAbilityList.ts](../src/hook/useAbilityList.ts) - `getAbilityListPaginated`
4. [src/hook/usePokemonByAbility.ts](../src/hook/usePokemonByAbility.ts) - `getPokemonByAbility`

#### 개선 효과

```diff
- 각 파일마다 20줄의 페이지네이션 로직 중복
+ 2줄의 유틸 함수 호출로 대체

Before: 4개 파일 × 20줄 = 80줄
After:  1개 모듈 94줄 + 4개 × 2줄 = 102줄
순 증가: 22줄 (하지만 타입 안전성과 재사용성 크게 향상)
```

#### 사용 예시

```typescript
// Before
import { produce } from 'immer'
import { PokemonEdge } from '~/graphql/typeGenerated'

await fetchMore({
  updateQuery: (previousQueryResult, { fetchMoreResult }) => {
    if (!fetchMoreResult) return previousQueryResult

    return produce(previousQueryResult, (draft) => {
      draft.getPokemonList.edges = [
        ...previousQueryResult.getPokemonList.edges,
        ...fetchMoreResult.getPokemonList.edges,
      ]
      draft.getPokemonList.pageInfo = {
        ...previousQueryResult.getPokemonList.pageInfo,
        endCursor: fetchMoreResult.getPokemonList.pageInfo.endCursor,
        hasNextPage: fetchMoreResult.getPokemonList.pageInfo.hasNextPage,
      }
    })
  },
})

const pokemonList =
  data?.getPokemonList?.edges?.map((edge: PokemonEdge) => edge.node) ||
  initialList

// After
import {
  mergePagedResults,
  extractNodesFromEdges,
} from '~/module/graphqlPagination.module'

await fetchMore({
  updateQuery: (prev, { fetchMoreResult }) =>
    mergePagedResults('getPokemonList', prev, fetchMoreResult),
})

const pokemonList = extractNodesFromEdges(
  data?.getPokemonList?.edges,
  initialList,
)
```
