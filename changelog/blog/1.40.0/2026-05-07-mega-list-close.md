# 메가진화 일람 페이지 신설(C-5) 종료 결정

> **결정 날짜**: 2026-05-07
> **브랜치**: `feature/1.40.0-mega-list-close`

## 결정 개요

**결정 유형**: 작업 종료 (코드 변경 없음)
**담당**: Claude + 사용자 검토
**연관 분석**: `.claude/research/reports/STR-2026-05-04-poke-korea-traffic-growth.md` (단계 C-5)

## 배경

C-5는 STR 보고서의 MT-4 항목("네이버 메가진화 키워드 강세 대응")을 근거로 별도 메가진화 일람 페이지(`/mega-evolution`) 신설을 계획한 작업이었습니다. 그러나 작업 착수 전 사용자가 "현재도 메가진화를 검색하면 메가진화가 활성화된 포켓몬 상세 페이지가 결과에 노출되는데, 굳이 별도 페이지가 필요한가?"라는 합리적 의문을 제기했고, 이를 검증하는 과정에서 작업이 불필요하다고 판단했습니다.

## 점검 결과

### 1. 사이트맵에 이미 등록되어 있음

`src/app/sitemap.ts`를 확인한 결과, 메가진화 일람 URL과 개별 메가진화 페이지가 모두 등록되어 있었습니다.

| URL | 라인 | priority |
| --- | --- | --- |
| `/list?isMega=true` | 298 | 0.8 |
| `/list?isRegion=true` | 304 | 0.8 |
| `/list?isGigantamax=true` | 310 | 0.8 |
| `/detail/{id}/mega` (전체) | 205-210 | 0.7 |

### 2. 동적 메타데이터가 자동 생성됨

`src/app/list/_metadata/generateListMetadata.ts:124-130, 160`에서 `?isMega=true` 쿼리를 감지해 다음 메타를 자동 생성하고 있었습니다.

- title: `메가진화 포켓몬 도감 | 포케 코리아`
- description: `메가진화 포켓몬을 한눈에 확인하세요. 메가진화 포켓몬을 빠르게 찾아보세요.`
- canonical: `https://poke-korea.com/list?isMega=true`

복합 필터(`?isMega=true&type=fire` 등)에 대해서도 자동으로 적절한 메타가 생성됩니다.

### 3. 개별 메가진화 상세 페이지의 SEO 대응 완료

단계 B-2에서 `/detail/{id}/mega` 메타 description을 자연어 패턴으로 정비한 작업(`generateChampionsDetailMetadata.ts` 등)이 이미 진행되어, "메가뮤츠", "메가리자몽" 같은 구체 키워드 인입은 개별 상세 페이지가 담당하고 있습니다.

## 별도 페이지 신설을 보류한 추가 사유

### 카니발리제이션 우려

`/mega-evolution`(신설)과 `/list?isMega=true`(기존)는 콘텐츠가 사실상 동일합니다. 검색엔진이 두 URL을 경쟁 관계로 인식하면:

- 어느 페이지를 우선 노출할지 혼란
- 같은 검색 의도에 대한 SEO 권위가 두 URL로 분산
- 통계상 노출수·클릭수가 분리되어 효과 측정도 어려움

### 기존 동적 메타 시스템과 충돌

별도 페이지는 `generateListMetadata.ts`의 자동 메타 생성 로직 외부에 두어야 하므로, 새로운 메타 관리 흐름이 추가됩니다. 향후 메타 정책 변경 시 두 곳을 동시에 수정해야 하는 부담이 발생합니다.

### "메가진화 일람" 검색 의도 충족

이 키워드 검색자는 "어떤 포켓몬이 메가진화하는지 모두 보고 싶음"이라는 의도를 가집니다. `/list?isMega=true`가 이미 이 의도를 완전히 충족하고 있습니다.

## 결론

별도 페이지 신설보다 현재 자산을 유지하는 편이 SEO와 유지보수 측면에서 모두 합리적입니다. STR MT-4의 "메가진화 키워드 강세 대응"은 현행 자산으로 이미 달성된 상태로 판단합니다.
